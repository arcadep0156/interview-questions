## Core Files

### Data Files

- `data/YYYY/company.json` - Question files organized by year and company
  - Each file contains company, year, and questions array
  - All questions have required fields per schema
  - JSON format with validation

### Metadata Files

- `meta/schema.json` - JSON Schema for validation
  - Defines required fields and types
  - Validates experience levels, difficulty
  - Ensures contributor object structure
- `meta/topics.json` - Approved topics list (27 topics)
- `meta/companies.json` - Company metadata (name, slug, category)

### Auto-Generated Files

- `index.json` - Master index of all questions
  - Generated by `scripts/generate-index.js`
  - Contains metadata: total questions, companies, years, topics
  - Lists all data files with paths
- `contributors.json` - Contributor leaderboard
  - Generated by `scripts/generate-contributors.js`
  - Includes name, github, linkedin, question count
  - Sorted by contribution count

### Documentation

- `README.md` - Project introduction with stats and top contributors
- `CONTRIBUTING.md` - Detailed contribution guidelines
  - Quick start (TL;DR)
  - Step-by-step instructions
  - JSON formatting rules
  - Topic and experience level standards
  - PR process
- `QUICK_REFERENCE.md` - Quick reference for contributors
- `docs/GITHUB_ACTIONS_SETUP.md` - CI/CD configuration

### GitHub Integration

- `.github/workflows/validate.yml` - Schema and duplicate validation
- `.github/workflows/auto-generate-index.yml` - Auto-generate index and contributors
- `.github/workflows/trigger-deploy.yml` - Trigger website rebuild


## JSON Schema

```json
{
  "company": "Amazon",
  "year": 2025,
  "questions": [
    {
      "id": "amazon-2025-001",
      "company": "Amazon",
      "year": 2025,
      "role": "DevOps Engineer",
      "experience": "3-5 years",
      "topic": "Docker",
      "question": "Question text",
      "difficulty": "medium",
      "contributor": {
        "name": "Full Name",
        "github": "@username",
        "linkedin": "https://linkedin.com/in/profile"
      },
      "contributedAt": "2025-02-20",
      "tags": ["docker", "containers"]
    }
  ]
}
```

### Field Specifications

| Field | Type | Format | Example |
|-------|------|--------|---------|
| id | String | company-year-number | amazon-2025-001 |
| company | String | Official name | Amazon, Google |
| year | Number | YYYY | 2025, 2024 |
| role | String | Job title | DevOps Engineer |
| experience | String | Standard level | Fresher, 3-5 years |
| topic | String | From topics.json | Docker, Kubernetes |
| question | String | Question text | "Explain..." |
| difficulty | String | easy/medium/hard | medium |
| contributor | Object | name, github, linkedin | See above |
| contributedAt | String | YYYY-MM-DD | 2025-02-20 |
| tags | Array | Keywords | ["docker"] |

## Contribution Workflow

### File Modification Pattern
1. Fork repository
2. Clone fork locally
3. Create feature branch
4. Edit `data/YYYY/company.json` (or create new)
5. Add questions with proper JSON formatting
6. Commit (only data files, not index/contributors)
7. Push to fork
8. Create Pull Request
9. Automated validation runs
10. Once merged, index and contributors auto-update

### Quality Gates
- JSON schema validation
- Duplicate ID detection
- Topic validation against approved list
- Manual review by maintainers

## Conventions

### File Naming
- Data files: `data/YYYY/company.json` (lowercase company name)
- Documentation: `UPPERCASE.md` for root-level
- Guides: `lowercase-with-hyphens.md` for subdirectories

### JSON Formatting
- Valid JSON syntax
- Unique IDs (company-year-number)
- Year as number, not string
- All required fields present
- UTF-8 encoding

### Git Workflow
- Branch names: `add-<company>-<topic>-questions`
- Commit messages: `Add: <description>`
- PR titles: `Add: [Company] [Topic] interview questions`

## Validation Rules

### Automated Checks
- JSON schema compliance
- Unique question IDs
- Valid topics from approved list
- Valid experience levels
- Valid difficulty levels
- Contributor object structure

### Data Quality
- Real interview questions only
- Clear, grammatically correct
- Proper context when needed
- Accurate company names
- Recent questions preferred (2023+)

## Integration Points

### GitHub Actions
- **validate.yml**: Schema validation, duplicate detection
- **auto-generate-index.yml**: Generates index.json and contributors.json
- **trigger-deploy.yml**: Triggers community-website rebuild

### Community Website
- URL: https://arcadep0156.github.io/community-website/interview-questions
- Data source: Raw JSON from GitHub
- Features: Search, filter, pagination
- Auto-rebuild: Triggered on data changes
- Contributors page: Leaderboard with LinkedIn

### Automation Flow
1. Contributor adds question → PR created
2. Validation runs → Schema check, duplicate check
3. PR merged → Auto-generate runs
4. index.json and contributors.json updated
5. Trigger-deploy runs → Website rebuilds
6. New questions appear on site (3 minutes)
